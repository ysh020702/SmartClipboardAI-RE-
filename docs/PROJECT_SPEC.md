# SmartClipboardAI 프로젝트 명세

## 프로젝트 목적

SmartClipboardAI는 사용자가 일상적으로 모으는 사진, 스크린샷, 복사한 텍스트, 복사한 링크를 하나의 데이터 흐름으로 정리하고, 이를 작업 주제인 `Topic`과 AI Agent의 실행 가능한 초안으로 연결하는 Android 앱입니다.

앱의 목표는 단순 저장이 아니라, 흩어진 정보를 사용자의 다음 작업으로 이어주는 것입니다.

## 핵심 사용자

- 자료를 여러 앱에서 모으는 대학생, 팀 프로젝트 참여자, 리서치 사용자
- 회의, 여행, 과제, 일정 준비처럼 여러 자료를 한 주제로 묶어야 하는 사용자
- 링크, 캡처, 짧은 메모를 모아 나중에 정리하는 사용자
- AI가 바로 실행하기보다 초안을 만들고 사용자가 검토하는 흐름을 원하는 사용자

## 핵심 문제

- 복사한 텍스트와 링크는 시간이 지나면 맥락을 잃기 쉽습니다.
- 스크린샷과 사진은 갤러리에 묻히고 작업 단위로 정리되지 않습니다.
- Android 정책상 백그라운드에서 몰래 클립보드나 화면을 감시할 수 없습니다.
- AI가 바로 외부 앱에 실행하면 사용자가 통제권을 잃습니다.

## 앱이 해결하는 방식

1. 사용자의 명시적 액션으로 데이터를 수집합니다.
2. 모든 입력을 `DataItem`으로 저장합니다.
3. `DataItem`에 타입, 출처, 시간, cluster 정보를 붙입니다.
4. 사용자가 최종 `Topic`을 선택하거나 생성합니다.
5. Gemini 기반 Agent가 `TopicAnalysis`와 `TopicAction` 초안을 만듭니다.
6. 사용자가 초안을 검토하고 수정한 뒤 외부 앱으로 전달합니다.

## 이번 버전 MVP 범위

- Topic/Agent 초안 경험을 1순위로 구현합니다.
- 링크/텍스트/이미지/스크린샷을 `DataItem`으로 저장합니다.
- 새 MediaStore 항목은 자동 확정 저장이 아니라 검토 후 저장합니다.
- `DataItem` / `DataItemEntity`에 cluster 필드를 추가합니다.
- Gemini Agent 계약과 초안 생성 흐름을 정의합니다.
- Samsung Notes는 공유 초안 전달, Calendar는 insert intent 초안 전달로 시작합니다.
- 기존 수집 구현은 가능한 한 재사용합니다.

## MVP 구현 우선순위

1. 협업 기준 문서와 현재 코드 감사 결과를 확정합니다.
2. 공통 모델, DB, Repository, Navigation, Permission/Manifest baseline을 먼저 정리합니다.
3. 데이터 수집 흐름은 Share, Tile, MediaStore, SAF로 나눠 기능별 브랜치에서 진행합니다.
4. UX는 Topic/Agent 초안 경험을 중심에 두고, 기존 `MainScreen`은 참고하되 화면 단위로 다시 나눕니다.
5. Gemini, OCR, OG 추출, cluster manager는 `DataItem` metadata 방향과 저장 흐름이 확정된 뒤 연결합니다.

현재 프로젝트 오너 확인 기준으로 `T-000-current-code-audit`와 `T-010-agents-and-docs-setup`은 완료된 기준 문서로 사용합니다. 다음 문서 작업은 `T-020-architecture-baseline`입니다.

## 이번 버전에서 하지 않을 것

- 백그라운드 클립보드 지속 감시
- 화면 자동 감시
- 접근성 API를 통한 무단 수집
- AI가 사용자 승인 없이 외부 앱에 바로 실행
- 클립보드 히스토리 전체 접근
- 완전 자동 분류만으로 Topic 확정
- 대규모 UI 리팩토링을 하나의 PR에 몰아넣기
- 별도 Cluster 테이블 설계

## 주요 입력 데이터

- 직접 찍은 사진
- 스크린샷
- 복사한 텍스트
- 복사한 링크

## 주요 결과물

### DataItem

모든 입력의 공통 저장 단위입니다. 텍스트, 링크, 이미지, 파일, 스크린샷을 표현합니다.

### Topic

사용자가 최종 확정한 작업 주제입니다. 여러 `DataItem`과 연결됩니다.

### TopicAnalysis

Agent가 주제와 자료를 분석한 중간 결과입니다. 요약, 핵심 포인트, 원본 데이터 ID를 포함합니다.

### TopicAction

사용자가 검토할 수 있는 실행 초안입니다. 요약, 일정, 리마인더, 공유 초안, 할 일 같은 타입을 가집니다.

## 사용자 검토 원칙

AI Agent는 사용자를 대신해 초안을 만들 수 있지만, 최종 실행은 사용자가 검토한 뒤 명시적으로 수행합니다. 이 원칙은 Notes, Calendar, Reminder, 공유 초안 모두에 적용합니다.
